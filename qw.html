在面向对象的编程中，接口是一种规范的定义，它定义了行为和动作的规范，在程序设计里面，接口起到了一种限制和规范的作用，接口定义了某一批类所需要遵守的规范，接口不关心这些类的内部状态数据，也不关心这些类里方法的实现细节，它只规定这批类里必须提供某些方法，提供方法的这些类就可以满足实际需要。

TypeScript中的接口类似于Java，同时还增加了更加灵活的接口类型，包括属性、函数、可索引和类等。

1. TypeScript中定义属性接口

  1. // 约束传入的对象
  2. interface FullName{
  3.     firstName:string;
  4.     secondName:string;
  5. }
  6.  
  7. // 传参的类型为约束好的接口对象
  8. function printName (name:FullName){
  9.     console.log(name.firstName+'--'+name.secondName);
  10. }
  11.  
  12. // 如果直接将参数传入，就只能传入约束过的字段值(firstName,secondName);
  13. printName({
  14.     firstName:'李',
  15.     secondName:'四'
  16. });
  17.  
  18. // 如果需要添加一些参数，需要将所有参数封装成一个对象传入。
  19. var obj = {
  20.     age:20,
  21.     firstName:'张',
  22.     secondName:'三'
  23. }
  24. printName(obj);

 也可以对接口中的属性定义是否可选。

  1. // 约束传入的对象
  2. interface FullName{
  3.     firstName:string;
  4.     // 如果添加了问号，表示参数可传可不传
  5.     secondName?:string;
  6. }
  7.  
  8. // 传参的类型为约束好的接口对象
  9. function printName (name:FullName){
  10.     console.log(name);
  11. }
  12.  
  13. // 由于secondName为可传可不传，所以只需要传入一个参数即可。
  14. printName({
  15.     firstName:'李四'
  16. });

2. TypeScript中定义函数类型接口。对方法传入的参数，以及返回值进行约束。

  1. // 定义一个加密的函数类型接口
  2. interface encrypt{
  3.     // 定义参数类型与返回类型
  4.     (key:string,value:string):string
  5. }
  6. // 参数类型与返回类型必须符合接口规范
  7. var MD5:encrypt = function(key:string,value:string):string{
  8.     // 模拟操作
  9.     return key + value;
  10. }
  11. console.log(MD5('username','password'));

3. TypeScript中定义可索引类型接口。实现对数组和对象的约束。

  1. // 对数组的约束
  2. interface UserArr{
  3.     // 约束索引为数字类型，数组中的元素为字符类型
  4.     [index:number]:string
  5. }
  6.  
  7. // var arr:UserArr = [123,456];
  8. // 报错
  9.  
  10. var arr:UserArr=['Lucy','Augus'];
  11. // OK

也可以实现对对象进行约束，用的比较少。

  1. // 对象的约束
  2. interface UserObj{
  3.     [index:string]:string
  4. }
  5.  
  6. // var arr:UserObj={age:20};
  7. // 报错
  8.  
  9. var arr:UserObj={age:'20'};
  10. // OK

4. TypeScript中定义类类型的接口。实现对类的约束。

  1. // 定义类的属性与方法 
  2. interface Animal{
  3.     name:string;
  4.     eat(str:string):void;
  5. }
  6. class Dog implements Animal{
  7.     name:string;
  8.     constructor(name:string){
  9.         this.name = name;
  10.     }
  11.     // 可以不传参数，但是必须要定义方法
  12.     eat(food:string){
  13.         console.log(this.name +'吃'+food);
  14.     }
  15. }
  16. var dog = new Dog('小黑');
  17. dog.eat('肉');

5. TypeScript中接口的扩展。实现接口的继承。

  1. // 定义父类接口
  2. interface Animal{
  3.     eat():void;
  4. }
  5.  
  6. // 定义子类接口
  7. interface Person extends Animal{
  8.     work():void;
  9. }
  10.  
  11. class Web implements Person{
  12.     public name:string;
  13.     constructor(name:string){
  14.         this.name = name;
  15.     }
  16.     // 必须实现父类接口Animal定义的方法
  17.     eat(){
  18.         console.log(this.name+'吃肉');
  19.     }
  20.     // 必须实现子类接口Person定义的方法
  21.     work(){
  22.         console.log(this.name +'写代码')
  23.     }
  24. }
  25.  
  26. var boy = new Web('小李');
  27. boy.eat();
  28. // 小李吃肉

接口在实现接口约束的同时还可以实现类的继承。

  1. // 定义父类接口
  2. interface Animal{
  3.     eat():void;
  4. }
  5. // 定义子类接口
  6. interface Person extends Animal{
  7.     work():void;
  8. }
  9. // 定义一个类
  10. class Programmer{
  11.     public name:string;
  12.     constructor(name:string){
  13.         this.name =name;
  14.     }
  15.     coding(code:string){
  16.         console.log(this.name + code);
  17.     }
  18. }
  19. // 继承一个类，并实现父类与子类的接口方法
  20. class Web extends Programmer implements Person{
  21.     constructor(name:string){
  22.         super(name)
  23.     }
  24.     // 实现基类Animal定义的方法
  25.     eat(){
  26.         console.log(this.name+'吃肉');
  27.     }
  28.     // 实现Person类定义的方法
  29.     work(){
  30.         console.log(this.name +'工作');
  31.     }
  32. }
  33.  
  34. var boy = new Web('小李');
  35.  
  36. boy.eat();
  37. // 小李吃肉
  38.  
  39. boy.coding('写前端代码');
  40. // 小李写前端代码
